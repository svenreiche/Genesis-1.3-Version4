# GENESIS 1.3 - Development

* Small correction for bugs and errors as well as cosmetic changes  can be committed directly to the master branch
* There is also dev branch as reference as a starting point of new development, though it is recommended to use a dedicated branch
* Addressing specific issue-tickets should be done on a dedicated branch, e.g. Issue-334
* Smaller addition should have their dedicated branch
* Projects (large scale addition) should be done and tested with new codes before replacing existing routines.
